Quarksql

Quarksql is a lightweight, embeddable SQL-like engine for C++. It layers SQL-style querying on RocksDB, adds in-memory indexing for speed, exposes a JWT-protected REST API with Crow, and lets you write business logic in JavaScript via an embedded V8 engine.

What it does

  • SQL: SELECT/INSERT/UPDATE/DELETE/BATCH, with WHERE (AND/OR), LIKE, ranges, JOIN, GROUP BY, ORDER BY, COUNT(*), SKIP/LIMIT
  • Storage: one RocksDB column family per table
  • Indexing: in-memory indices for fast equality filters & joins
  • Pagination: SKIP/LIMIT pushed into RocksDB iteration
  • Server: Crow HTTP + JWT auth; interactive console at /
  • JavaScript: V8 executes scripts/business.js, auth.js, sanitize.js

Install & Build (Ubuntu 22.04)

sudo apt update
sudo apt install -y build-essential cmake librocksdb-dev libv8-dev libv8-headers libssl-dev nlohmann-json3-dev

git clone https://github.com/lucpattyn/quarksql.git
cd quarksql
mkdir build && cd build
cmake .. -DCMAKE_CXX_STANDARD=17
cmake --build . -- -j$(nproc)

Prepare the build directory

Copy into build/ after compiling:
• schemas.json
• public/ (contains index.html)
• scripts/ (your V8 JS modules)

Run

./quarksql-server --dbpath ./data --port 18080 --jwt-secret mysecret

Open http://localhost:18080/ for the SQL console.

Supported SQL (examples)

-- SELECT
SELECT * FROM users;
SELECT * FROM products WHERE price > '20';
SELECT * FROM journal WHERE date >= '2025-01-01';
SELECT * FROM products ORDER BY stock DESC SKIP 1 LIMIT 3;

SELECT COUNT(*) FROM orders;
SELECT user, COUNT(*) FROM orders GROUP BY user ORDER BY COUNT DESC;
SELECT productId, COUNT(*) FROM orders GROUP BY productId ORDER BY COUNT DESC SKIP 0 LIMIT 5;

SELECT orders.id, users.email FROM orders JOIN users ON orders.user = users.email;
SELECT orders.id, products.name, orders.qty FROM orders JOIN products ON orders.productId = products.id;

-- INSERT
INSERT INTO users VALUES {"email":"alice@example.com","password":"secret"};
INSERT INTO users (email,password) VALUES ('alice@example.com','secret');

-- UPDATE
UPDATE users SET {"password":"newsecret"} WHERE email='alice@example.com';
UPDATE users SET password='newsecret' WHERE email='alice@example.com';

-- DELETE
DELETE FROM users WHERE email='alice@example.com';
DELETE FROM users KEYS ["alice@example.com"];

-- BATCH
BATCH products {
  "p1":{"id":"p1","name":"Widget","price":19.99,"stock":100},
  "p2":{"id":"p2","name":"Gadget","price":29.99,"stock":50}
};

Business Logic (V8 + JS)

The server embeds V8 and loads scripts/business.js, auth.js, and sanitize.js at startup. C++ exposes helpers like CppQuery, CppExecute, CppSignJwt, CppVerifyJwt into the JS runtime. business.js builds an api object consumed by the HTTP routes. 

// business.js (simplified)
api.login = (username, password) => ({ token: CppSignJwt(JSON.stringify({ sub: username })) });
api.verify = (token) => ({ valid: true, data: JSON.parse(CppVerifyJwt(token)) });
api.query = (sql) => JSON.parse(CppQuery(sql));      // SELECT
api.execute = (sql) => JSON.parse(CppExecute(sql));  // INSERT/UPDATE/DELETE/BATCH

REST API

  • POST /api/login ? returns JWT
  • POST /api/verify ? validates JWT
  • POST /api/query ? run SELECT
  • POST /api/execute ? run INSERT/UPDATE/DELETE/BATCH
